Genetic Algorithm

This is a Javascript implementation of a genetic algorithm inspired by Richard Dawkin's Weasel Program, an excellent demo of which you can find here.

Above, you can see two algorithms in action. The goal of each is to transform a randomly generated sequence of characters into the target phrase 'Methinks it is like a weasel'. You can pick your own target phrase by entering some text in to the input box and hitting 'Evolve!'.

The Random Algorithm's approach is simple. It generates random sequences of characters of the same length as the target phrase and checks to see if any of them match the target phrase exactly. If one does match, it stops - success! If not, it repeats the process, generating a new set of random characters and once again checking for an exact match. You can think of it as a set of instructions like this:

  1. Generate 300 random sequences of characters
  2. For each one, count the differences between the generated phrase and the target phrase
  3. If any of the counts are 0, finish.
  4. If none of the counts are 0, goto 1.

Step 2, where we count the differences, is often called getting the character sequences 'fitness'. The closer to the target a character sequence is, the more 'fit' we can consider it to be. In the algorithms above, the more 'fit' a population member is, the more green it's color.

The random algorithm continues ad infinitum, or more preciesly, it's likely to take around 700,227,000,000,000,000,000,000,000 years to generate a successfull match! (Asumming the algorithm generates and checks 300 new character sequences every 5ms.)

The genetic algorithm takes a more sophisticated approach. It too starts off with generating a set of random character sequences and counting the differences between them and the target phrase. But then, instead of throwing them all away and generating a new set if none of them match, the genetic algorithm orders the set of character sequences it just generated according to their fitness (how many differences there are between the sequence and the target phrase). Then, to generate a new set (or generation) of character sequences it combines the fittest members of the previous generation and with some small chance, randomly changes (or mutates) some of the characters in each sequence.

There are lots of different ways you can combine population members. These are usually called 'crossover strategies'. This example takes one character from the first sequence, then a character from the second, then another character from the first, etc. So a comination of the sequences 'aaa' and 'bbb' would be 'aba'.

As a set of instructions, the genetic algorithm looks like this:

  1. Generate a population of 300 random sequences of characters
  2. For each population member, count the differences between the generated phrase and the target phrase
  3. If any of the counts are 0, finish.
  4. If none of the counts are 0, order the generated phrases by their fitness
  5. Until you have 300 new members:
    1. Using a probability distribution which gives us a higher chance of picking a fitter member, pick two existing population members
    2. Combine those two population members to form a new population member
    3. With some small probability, change some of the characters in the new population member
  6. Goto 2

Because we only breed from the fittest memebers of the population each time, every generation brings us closer and closer to the target phrase. This approach is much, much faster. On average, it takes only a few hundred generations (which it can generate in seconds) to get to the target phrase.

As well as small projects like this, genetic algorithms have been used to design cars, electronic circuits, and evolve virtual creatures. It's also a neat demonstration of the effectiveness of evolution and natural selection.
